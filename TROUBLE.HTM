Trouble Shooting      

If Qlets doesn't start up when you attempt to run QLETS.EXE, you may have to tweak some settings. The most common problem is a memory conflict on older machines running Windows 3.1 or DOS. The following section is reprinted from the User Manual (QMANUAL.DOC), that comes bundled in the download zipfile.

The single persistant problem with QLETS revolves around corruption of index files, extension .MDX or .NDX. Theses files appear in virgin form in the download copy after it's unpacked (by clicking on the download file, which is self-extracting).

Corrupted index files can cause the appearance of incorrect ads, trades, or memo/list information attributed to accounts, and is the first indication of a problem. First run Reindex under Special Tools and look for an error message. Almost always, such files are corrupted due to an improper shutdown, a hardware conflict, or another circumstance that requires rebooting the computer while QLETS is still running. This event can cause imcomplete records that have a variety of random characters, and may be missing key information needed by the index files, depending on where the program code was waiting and which database files may have been open at that point.

It is possible to repair all corrupted index files by using the following instructions:

1) move all of your old QLETS files into a directory called C:\QL17 (you may have to create it). This directory is read first when searching for older QLETS copies using the Convert Data option (below), to extract their current data into clean files.
2) download the most current Full Version of QLETS from our website and save it in C:\QL8
3) click on the download file (Q8FULL.EXE), to unpack it into C:\QL8
4) run the newly downloaded version at C:\QL8\QLETS.EXE
5) Use the Convert Data option under Special Tools (if you have problems with the password, the default is a blank line so hit Enter). Then choose QLETS - Earlier Versions by hitting Q.
6) Your current data is then extracted from C:\QL17 into clean files properly indexed.

 About the Virtual Memory Manager

A virtual memory manager (VMM) is included with QLETS to help it to run concurrently with other software in Windows 3.1. By using the VMM, memory requirements for Qlets are much reduced. The VMM is currently set for a typical 4 RAM machine (you can edit the settings by opening QLETS.VMM in a text editor), but can be set for as little as 2 RAM. It may conflict with disk-caching software like SMARTDRIVE, however, a common disk-cache that may well be running on your system. (SMARTDRIVE.EXE is shipped with Microsoft Windows). If you are unable to start up QLETS, it's likely due to this memory conflict. There are two ways to fix it, should this happen to you. You can:

1) disable SMARTDRIVE (or whatever disk-caching program is running on your system), or
2) disable the VMM supplied with QLETS (by erasing the QLETS.VMC)


 Disabling the SMARTDRIVE Disk Cache

Disabling SMARTDRIVE involves editing your AUTOEXEC.BAT file, to convert the line that loads it into a comment. For example, my AUTOEXEC file contained the line:

C:\WINDOWS\SMARTDRV.EXE 1024 256       which was changed to:
REM C:\WINDOWS\SMARTDRV.EXE 1024 256

It's useful, but not essential, to edit the QLETS.VMC file, (you can use any text editor, like Notepad, for example) which contains explanations of the settings. If the QLETS.VMC file is present - ie. not erased - then the VMM will be activated whenever Qlets is started. The MAXMEM and MINMEM settings in the VMM can be altered to suit your RAM. For information on SMARTDRIVE, type "help" from the c:\ and look at the entry describing it. The downside of disabling SMARTDRIVE (and using the Qlets VMM instead), is slightly slower computer operation, especially when groups of files are copied or moved. The upside is you'll be able to use Alt-Tab in Windows to switch between other software and Qlets while keeping them all running at once.


 Disabling the Qlets Virtual Memory Manager

The other option is to keep SMARTDRIVE in place, and remove the file QLETS.VMC from the C:\QL8 directory instead. This will disable the Qlets virtual memory manager. You must then add the following line to your CONFIG.SYS file:

     SET DOS16M=:2M

This is a DOS environmental variable that sets how much extended memory is available for DOS-based programs. For users with little RAM, disabling the VMM this way means that Qlets must run alone from the DOS prompt.
